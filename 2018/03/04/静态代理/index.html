静态代理

Posted on 2018-03-04 | Edited on 2018-06-04 | In Spring | Views:

代理模式/proxy可以分为三部分：调用端，代理类，目标类。
根据开闭原则：对修改封闭，对开放扩展。假设有一个类，其中方法是写好的现成业务逻辑，现在需要加一些业务无关的事情例如记录这个业务方法的执行时间，按照开闭原则，可以通过代理模式调用目标方法，在代理类的方法里对它的实现扩展，而不是直接修改这个方法。

静态代理例子：

1.代理类和目标类实现同一个接口
2.代理类里有一个目标类对象
3.代理类会在自己方法里扩展逻辑，然后调用目标类的方法
4.调用方只需要创建目标类对象，用这个构造代理类，然后调用代理类方法即可
5.这样扩展逻辑可以全部写在代理类中
6.但是如果接口改动，目标类和代理类就都需要修改

接口：
public interface Target {
    void doSomething();
}

目标类：
public class TargetImpl implements Target{
public void doSomething(){
    for(int i=0;i<3;i++) {
            new Thread(() -> System.out.println(Thread.currentThread().getName() + " is running!")).start();
        }
    }
}

代理类：
public class Proxy implements Target{
private Target target;

public Proxy(Target target){
    this.target=target;
}

@Override
public void doSomething() {
    long start=System.currentTimeMillis();
    target.doSomething();
    long end=System.currentTimeMillis();
        System.out.println("execution time is "+(end-start)+"ms");
    }
}

调用类：
public class Client {
public static void main(String[] args) {
    Target target=new TargetImpl();
    Proxy proxy=new Proxy(target);
    proxy.doSomething();
    }
}

输出（只开启3个线程异步执行，开3个线程用了53ms）：
Thread-0 is running!
execution time is 53ms
Thread-2 is running!
Thread-1 is running!
