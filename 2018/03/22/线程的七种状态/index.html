线程的七种状态

Posted on 2018-03-22 | Edited on 2018-05-22 | In Java并发编程 | Views:

new:新建状态，new一个thread或者它的子类

ready to run:

  1. 调用start()，但没有拿到时间片，等待CPU资源
  2. 已经获取时间片进入run()方法的逻辑，但是用完后从running变成ready to run
  3. 已经running的线程，让出CPU资源，进入blocked状态，例如Thread中的sleep（）方法，时间到了后再次进入ready to run

running：运行状态，ready to run的线程获取了CPU资源

waiting：Object的wait方法（让出CPU资源并且放锁，需要其他线程notify），Thread的join()方法等待其他线程，或者park()方法

sleeping：睡眠状态，调用线程的sleep()方法（让出CPU资源方便其他线程运行，但是不释放锁，等待sleep方法时间到自动唤醒）

blocked：阻塞状态，线程等待获取锁

dead：

  1. run()方法逻辑结束，运行完毕
  2. 线程响应中断
  3. 线程上层的进程退出
