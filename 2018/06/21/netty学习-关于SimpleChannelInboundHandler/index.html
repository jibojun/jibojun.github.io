netty学习:关于SimpleChannelInboundHandler

Posted on 2018-06-21 | Edited on 2018-07-11 | In Netty | Views:
  1. 使用channelRead0方法，重写其中逻辑，这个方法是一个抽象方法。
  2. SimpleChannelInboundHandler继承了ChannelInboundHandlerAdapter。
  3. 主要重写了channelRead方法，简单逻辑就是如果消息是SimpleChannelInboundHandler<>中指定的类型（acceptInboundMessage判断，需要注意如果之前还有handler，需要保证给SimpleChannelInboundHandler的数据类型符合），就会调用channelRead0方法。否则，就给下一个handler，channelRead0方法不执行，所以当消息类型不匹配时，我们在这个方法内写的逻辑无法执行。
  4. 可以看到finally中已经做好了消息释放，所以要注意后面不能再有handler接收SimpleChannelInboundHandler处理后的消息了。

  5. 从Netty4以上版本注释中可以看出在netty5中，channelRead0将会被改名为messageReceived。

        /**
 * Returns {@code true} if the given message should be handled. If {@code false} it will be passed to the next
 * {@link ChannelInboundHandler} in the {@link ChannelPipeline}.
 */
public boolean acceptInboundMessage(Object msg) throws Exception {
    return matcher.match(msg);
}

@Override
public void channelRead(ChannelHandlerContext ctx, Object msg) throws Exception {
    boolean release = true;
    try {
        if (acceptInboundMessage(msg)) {
            @SuppressWarnings("unchecked")
            I imsg = (I) msg;
            channelRead0(ctx, imsg);
        } else {
            release = false;
            ctx.fireChannelRead(msg);
        }
    } finally {
        if (autoRelease && release) {
            ReferenceCountUtil.release(msg);
        }
    }
}

/**
 * <strong>Please keep in mind that this method will be renamed to
 * {@code messageReceived(ChannelHandlerContext, I)} in 5.0.</strong>
 *
 * Is called for each message of type {@link I}.
 *
 * @param ctx           the {@link ChannelHandlerContext} which this {@link SimpleChannelInboundHandler}
 *                      belongs to
 * @param msg           the message to handle
 * @throws Exception    is thrown if an error occurred
 */
protected abstract void channelRead0(ChannelHandlerContext ctx, I msg) throws Exception;
