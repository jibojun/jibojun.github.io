线程池awaitTermination

Posted on 2018-06-01 | Edited on 2018-06-17 | In Java并发编程 | Views:

有时应用场景是需要一些东西并发处理，但并非异步，主线程要等待多线程并发操作完成，然后继续进行下一步。使用同步辅助类例如CountDownLatch（创建执行线程数量的闭锁，主线程等待，执行线程count down），CyclicBarrier（创建主线程+执行线程数的栅栏，全部awiat），线程join方法都可以做到。但如果使用线程池的话，可以依赖线程池的awaitTermination方法来使主线程等待直到全部任务全部执行完毕。

这个方法有以下特点：

  1. 必须在线程池关闭后调用（shutDown或者shutDownNow），一直阻塞直到完成。
  2. 方法需要两个参数，一个是超时时间，另一个是超时时间的单位，如果超时就会监控此时线程池是否关闭，未关闭返回false，关闭返回true。
  3. 可以超时时间设一个很大的值，确保线程池任务执行不会超出，在之前调用shutDown(等待任务完成退出)，然后调用awaitTermination，这样可以阻塞，达到等待线程池所有任务完成的目的。
  4. 也可以设一个短的超时时间，调用awaitTermination根据返回值来判断此时线程池是否关闭。
  5. 综上，这是一个阻塞方法，三种条件触发，一是shutdown后任务执行完了，二是到了自己设的超时时间，直接判断此时线程池是否关闭，三是线程中断。

例子：

    public static void main(String[] args) throws InterruptedException {
    ExecutorService tp=Executors.newFixedThreadPool(5);
    long start=System.currentTimeMillis();
    for(int i=0;i<15;i++){
        tp.submit(new Thread(() -> System.out.println("a new thread is running")));
    }
    tp.shutdown();
    if(tp.awaitTermination(1,TimeUnit.SECONDS)){
        System.out.println("thread pool is closed");
    }else{
        System.out.println("thread pool hasn't been closed yet");
    }
    long end=System.currentTimeMillis();
    System.out.println("execution time is:"+(end-start)+"ms");
}
