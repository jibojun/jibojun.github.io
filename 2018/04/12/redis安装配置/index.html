redis安装配置

Posted on 2018-04-12 | Edited on 2018-08-12 | In Redis | Views:
  1. redis 可下载源码编译安装：http://redis.io/

  2. 解压缩并跑以下命令：

    sudo make test
sudo make install

  3. 进 usr/local/bin,发现redis脚本已生成

  1. 配置redis.conf文件作为redis配置，包含日志，绑定地址端口，持久化RDB,AOF等所有配置：

例子：

    #修改为守护模式

daemonize yes

#设置进程锁文件

pidfile /Library/redis/redis.pid

#端口

port 6379

#绑定主机名

bind 127.0.0.1

#客户端超时时间

timeout 300

#日志级别，四个级别，默认verbose

loglevel verbose

#日志文件位置

logfile /Library/redis/log-redis.log

#设置数据库的数量，默认数据库为0，可以使用SELECT <dbid>命令在连接上指定数据库id

databases 1

##指定在多长时间内，有多少次更新操作，就将数据同步到数据文件，可以多个条件配合

#save <seconds> <changes>

#Redis默认配置文件中提供了三个条件：

save 900 1

save 300 10

save 60 10000

#指定存储至本地数据库时是否压缩数据，默认为yes，Redis采用LZF压缩，如果为了节省CPU时间，

#可以关闭该#选项，但会导致数据库文件变的巨大

rdbcompression yes

#指定本地数据库文件名

dbfilename dump.rdb

#指定本地数据库路径

dir /Library/redis/db/

#指定是否在每次更新操作后进行日志记录，Redis在默认情况下是异步的把数据写入磁盘，如果不开启，可能

#会在断电时导致一段时间内的数据丢失。因为 redis本身同步数据文件是按上面save条件来同步的，所以有

#的数据会在一段时间内只存在于内存中

appendonly no

#指定更新日志条件，共有3个可选值：

#no：表示等操作系统进行数据缓存同步到磁盘（快）

#always：表示每次更新操作后手动调用fsync()将数据写到磁盘（慢，安全）

#everysec：表示每秒同步一次（折衷，默认值）

appendfsync everysec
  1. 按照配置文件启动服务器，redis-server ../etc/redis.conf

  2. 关闭redis，用client终端，redis-cli shutdown
