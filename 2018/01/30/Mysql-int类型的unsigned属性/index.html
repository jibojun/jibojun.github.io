Mysql int类型的unsigned属性

Posted on 2018-01-30 | Edited on 2018-09-02 | In 关系型数据库 | Views:
  1. mysql中int字段具有zerofill和unsigned属性，unsigned代表着无符号的意思。
  2. unsigned使Int类型变为非负数。
  3. 主要作用是扩充正数范围，不加unsigned的tinyint范围为-128到127，加了unsigned，则不能为负数，变成0-255。例如设置ID主键时经常使用
  4. 各种int类型加unsigned与不加unsigned的取值范围如下：
1. tinyint: 不加：-128到127 加：0到255 1字节
2. smallint: 不加：-32768到32767 加：0到65535 2字节
3. mediumint: 不加：-8388608到8388607 加：0到16777215 3字节
4. int: 不加：-2147683648到2147683647 加：0到4294967295 4字节
5. bigint: 不加：-9223372036854到9223373036854775807 加：0到18446744073709551615 8字节

  1. unsigned只针对整型，float和double精度类型和decimal（数字类型，无精度缺失）没有，任何时候有负数。

    1. float: 最小非零值：±1.175494351e - 38 4字节
2. double: 最小非零值：±2.2250738585072014e - 308 8字节
3. decimal: 取决于decimal(a,b)中的a,b值，a为小数点左右存储总长最大值，b为小数点右边数字个数最大值。 占用存储空间为a+2字节

  2. 因为整型设置了unsigned属性后只能为非负数，所以在需要存储负数场景时不能使用。
