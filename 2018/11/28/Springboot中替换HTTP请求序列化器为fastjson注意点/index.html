Springboot中替换HTTP请求序列化器为fastjson注意点

Posted on 2018-11-28 | In Spring | Views:
  1. Spring mvc中rest controller中请求默认会用jackson来做json序列化、反序列化。
  2. 实际应用中，可能需要按照需求替换序列化器。
  3. 替换时，重写configureMessageConverters方法，此方法的传参是converter列表，新建fastjson转换器实例，加入此列表中。
  4. 但是会发现Spring boot2中这个替换不生效，例如打上了fastjson注解@JSONFeild的属性收到HTTP请求时会转换出错。

  5. 这是因为converters是一个列表，spring mvc加载列表中所有转换器，按先后顺序匹配。虽然我们添加了fastjson转换器，但jackson转换器的顺序在fastjson之前。

            org.springframework.http.converter.json.MappingJackson2HttpMessageConverter@61a93e7c
org.springframework.http.converter.json.MappingJackson2HttpMessageConverter@2a0a9212
com.dreamer.core.converter.FastJsonJsonpHttpMessageConverter@6a400857

  6. 此处可以有几种处理方法，方法一是移除jackson转换器只保留fastjson转换器，处理converter列表，移除jackson转换器，可以遍历List，用instance of进行判断，找出MappingJackson2HttpMessageConverter然后移除出列表。或者新建一个converter列表代替原来的，将的原来的一个个加入，判断到jackson转换器则不加入。

  7. 经过测试，第二种处理方式是将HttpMessageConverters对象配置出一个bean(代码配置@Bean注解)以使它的顺序在列表中先于fastjson转换器。

    @Bean
public HttpMessageConverters fastJsonHttpMessageConverters() {
...
}

  8. 除开替换无效问题，还有一个问题是使用spring而不是springboot时也共有的中文乱码问题，只需要配置UTF-8为JSON格式就可以解决。假设使用spring时，在上下文XML里也可以进行相应配置。在springboot中设置fastjson转换器属性如下：

    List<MediaType> mediaTypes = new ArrayList<>();
mediaTypes.add(MediaType.APPLICATION_JSON_UTF8);、
fastJsonHttpMessageConverter.setSupportedMediaTypes(mediaTypes);
