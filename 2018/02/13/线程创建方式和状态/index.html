线程三种创建方式

Posted on 2018-02-13 | Edited on 2018-05-14 | In Java并发编程 | Views:
  1. 继承Thread类(实现了Runnable接口)，在run()方法中写线程业务逻辑。start（）方法启动线程，调用run（）方法是当前线程执行，不会起新线程异步执行。
  2. 实现Runnable接口（将实现接口的类实例传入Thread构造方法或者直接用匿名内部类），同样在run()中写逻辑，start（）方法启动。

例子，匿名类：

new Thread(new Runnable() {
            public void run() {
                System.out.println("do something");
            }
        }).start();

比较：

共同点：

  1. 两者都是无返回值的，run()方法是void的。
  2. 线程的启动顺序是固定的按代码，而具体执行顺序是不固定的，由JVM和OS决定。

不同点：

  1. Thread因为是类，不能多继承。Runnable是接口，可以多个类实现它
  2. Runnable和Callable可以作为task传入Executor线程池，更加方便管理。

组合使用：

Thread的子类可以传入不同实现了Runnable接口类实例。

可以看出run()方法是void的，如果线程做的事情是需要返回值的，就可以靠Callable接口，这是1.5 JUC中新增加的，十分类似，逻辑写在call()方法里。

public interface Callable<V> {
    /**
     * Computes a result, or throws an exception if unable to do so.
     *
     * @return computed result
     * @throws Exception if unable to compute a result
     */
    V call() throws Exception;
}
  1. call()方法有返回值，而且是泛型的。
  2. 配合ExecutorService线程池用，submit(）方法一般用来提交Runnable或者Callable。
  3. Callable可以配合JUC中的Future接口或者FutureTask类里的get方法来获取线程执行完的返回结果。
  4. 适用于一些异步计算业务场景，新起线程提交给线程池，调用get方法时阻塞，直到拿到执行完的结果。
