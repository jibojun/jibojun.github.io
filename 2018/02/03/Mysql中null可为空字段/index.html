Mysql中null可为空字段

Posted on 2018-02-03 | Edited on 2018-09-09 | In 关系型数据库 | Views:
  1. 使用关系型数据库时字段列可以选择是否允许NULL值。
  2. 有时一些业务字段可为空，例如前端让用户输入电话号码，但是是选填，用户可以不填，此时落库的这列电话号码为了方便则可能设为允许NULL。
  3. 但从MYSQL性能和业务数据列的规范性上，可为空字段可能较少使用。

  4. MYSQL NULL字段可能有以下问题：

    1. 索引不存储NULL值，会有一种场景，查询一行，其中某一列的值是NULL，此时WHERE中用了负向不等于查询！=,则返回不出这一行。其它负向条件例如NOT IN同理，容易出问题。
2. 聚合函数，例如字符串拼接CONCAT，统计数量COUNT，有NULL情况下CONCAT会始终返回NULL，COUNT这一列时无法统计值为NULL的数据。
3. NULL会造成索引问题，可为空的列会造成索引键值增加额外空间一个字节判断NULL是否为空（MyIsam）。
4. 比如业务场景一个选填字段，用户原来没填数据库里设了NULL，后来业务上要改成必填字段，则会有一个从NULL更新到NOT NULL的过程，这里无法原地更新，同时易造成索引问题。

  5. 所以Mysql中应尽量避免使用可为空的列，解决方法是没值情况下给一个业务上不可能插入的值作为NULL的标记。一定要使用时也要避免列上索引。

  6. Mysql官方说明：

    NULL columns require additional space in the rowto record whether their values are NULL. For MyISAM tables, each NULL columntakes one bit extra, rounded up to the nearest byte.
