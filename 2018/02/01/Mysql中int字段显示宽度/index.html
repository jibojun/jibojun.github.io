Mysql中int字段显示宽度

Posted on 2018-02-01 | Edited on 2018-09-02 | In 关系型数据库 | Views:

  1. Mysql中int()括号中长度并不影响数据存储，不同int类占的字节（1byte=8bits）分别如下：

    tinyint: 1字节
smallint: 2字节
mediumint: 3字节
int: 4字节
bigint: 8字节

  2. 而int(）括号中的值代表了显示宽度，最大有效显示宽度为255.

  3. 显示宽度指的是在填充0属性zerofill开启时，假设设为5，具体数字为1，则会从左侧以0填充变成00001。

  4. int列可以设unsigned和zerofill属性，例如：

    `id` INT( 11 ) UNSIGNED ZEROFILLNOT NULL AUTO_INCREMENT PRIMARY KEY

  5. int默认11.

  6. 有一些业务场景会用到显示宽度设置与0填充属性，例如：

    1. 月份列中10月前的月份，希望显示2位，则可以设宽度为2填充0，则2月为02。
2. 一些固定长度的ID,流水号，需要左侧填充0，则可以开始0填充并按照要求设置显示宽度
