Mybatis Sql中#{}与${}

Posted on 2018-02-19 | Edited on 2018-05-19 | In Mybatis | Views:

在mapper文件中，我们需要给dao接口里的方法编写sql，会接收来自方法中的传参。

传参变量可以用#{}或者${}，例如：

select * from table where id=#{id}

select * from table where id=${id}

Preparedstatement:预编译的SQL语句对象。SQL有预编译，在DB端内存可以缓存一些编译后的SQL。再次执行同样SQL就不需要预编译了，可以运行，提高性能。

statement:静态完整的SQL语句运行，并且负责返回结果。

$

  1. 是SQL中字符串替换，静态语句，直接参与了SQL的编译。对应statement。
  2. 无法避免SQL注入攻击。SQL注入的恶意语句是会一起被编译，最后被运行。

#

  1. 用于参数传递，对应PreparedStatement,预编译里参数会用占位符？代替，创建PreparedStatement,与SQL语句绑定，之后才填入具体参数。因为创建PreparedStatement时候已经传了SQL，执行时不再需要传入SQL。
    参数部分并不是SQL指令的一部分，DB做完SQL指令编译后，才用参数来执行，所以就算参数里含有SQL注入的恶意语句，最后也不会被运行。
  2. 可以避免大部分SQL注入攻击。
  3. 因为预编译SQL会被缓存在数据库，在数据库做参数化查询，性能会更好。

SQL注入例子：
select * from account where name=${name}

在name部分填入:

‘xxx’ or ‘1’=’1’

拼入恒等条件，查出数据。

拼入恒等条件后，可以用；结束这句语句，后面再跟其它破坏语句，例如DROP TABLE。
