JDK8的metaspace

Posted on 2018-07-02 | Edited on 2018-08-11 | In JVM | Views:
  1. Hotspot VM中的永久代，就是JAVA虚拟机规范中的方法区。
  2. 在JDK1.8之前一直存在，用来存储元数据，例如类信息，常量等。

  3. 在openjdk 1.8中已经废除了永久代，原因是与Oracle的JRockit兼容改成一样的。并且实际使用中，在目前动态代理使用较多情况下，永久代较容易出现OOM的问题。

    Motivation
    This is part of the JRockit and Hotspot convergence effort. JRockit customers do not need to configure the permanent generation (since JRockit does not have a permanent generation) and are accustomed to not configuring the permanent generation.

  4. 移除后，PermSize MaxPermSize已没有作用，如果配置的话则会被ignore option。

  5. 初始化和最大值参数变成例如-XX:MetaspaceSize=64m -XX:MaxMetaspaceSize=80m，最大值参数主要防止元空间过大影响本地服务器机器内存。

  6. 元空间的大小不是固定，而是动态变更的。一部分转入本地内存（例如class loader加载的类信息），一部分放入堆区（例如字符串常量）。

    Description
Move part of the contents of the permanent generation in Hotspot to the Java heap and the remainder to native memory.
  7. 因为动态增长，所以有MaxMetaspaceExpansion（默认5M）和MinMetaspaceExpansion（默认330K）参数来配置动态增长最大和最小幅度。
