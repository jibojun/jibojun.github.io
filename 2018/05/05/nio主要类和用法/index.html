nio主要类和用法

Posted on 2018-05-05 | Edited on 2018-07-07 | In Java IO | Views:

Selector:

  1. 相当于监听器，向上面注册CHANNEL。
  2. 通过select()方法获取就绪事件，没有就绪事件会阻塞，写在循环内轮询获取。

  3. select()后用selectedKeys()方法，返回值是SelectionKey对象的set，对于每个key可以通过以下方式判断就绪事件，也可以通过attachment拿取附加对象。

    selectionKey.isAcceptable();  
selectionKey.isConnectable();  
selectionKey.isReadable();  
selectionKey.isWritable();

  4. 遍历selectedKeys set时，需要显式在iterator里remove，否则会反复进入无法退出。

  5. interestOps()可以确定监听感兴趣的事件
  6. 关闭方法close()

Channel:

  1. 类似BIO的数据流，但是流是单向，CHANNEL双向。

  2. 需要把channel注册到selector上以便监听channel里是否有就绪事件，注册时需要选择感兴趣的事件，如下：

    SelectionKey.OP_CONNECT
SelectionKey.OP_ACCEPT
SelectionKey.OP_READ
SelectionKey.OP_WRITE　　

  3. nio channel有以下类型

    FileChannel：从文件中读写数据，不能非阻塞　　
DatagramChannel：通过UDP读写网络中的数据　　
SocketChannel：通过TCP读写网络中的数据　　
ServerSocketChannel：监听新进来的TCP连接，对每一个新进来的连接创建一个SocketChannel

  4. configureBlocking()设置是否阻塞，register()方法注册到selector，attach()方法附加对象

Buffer:

  1. 数据内存对象，根据基本类型有以下：

    ByteBuffer　　
MappedByteBuffer　　
CharBuffer　　
DoubleBuffer　　
FloatBuffer　　
IntBuffer　　
LongBuffer　　
ShortBuffer

  2. 服务端，channel的数据读到buffer，buffer的数据写入channel。

  3. 操作时，allocate()方法划分内存，put和get读写数据，flip切换读写模式，clear切换读写且后面数据覆盖前面数据，compact切换读写，把未读取数据放到头部，新写入数据在这部分数据之后。mark()方法写入位置标记,reset方法使位置到标记时（position属性反应了写数据时的位置，最大值为capacity-1）。
