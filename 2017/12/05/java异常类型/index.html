java异常类型

Posted on 2017-12-05 | Edited on 2018-07-07 | In JavaCore | Views:
  1. Throwable是Error和Exception的父类，catch Throwable可以抓住一切问题。
  2. Error是不被处理的问题，JVM本身错误为主(VirtualMachineError)，最典型的是Stackoverflow(栈溢出，例如递归无出口，反复压栈，溢出)，Out of MEMORY/oom，内存不足。Exception代表异常，所有异常是它的子类，catch Exception可以抓住所有类型异常。
  3. RuntimeException是Exception的子类，运行时异常，常见的如数组越界（访问不存在的下标位置），类没找到，空指针（调用一个Null对象的方法），一般是代码错误导致，不需要用try, catch处理。
  4. 在Exception中还一般分为checked exception和unchecked exception，检查异常是需要我们用try catch finally代码块去处理的，或者throw出去让更上层的调用方法去处理。在实际系统中，一般是一些运行环境造成的问题，可能出现或者不出现，出现了此类问题catch住后可以记录log以便排错，前端输出错误信息。最常见的有数据库操作的SQLException，IO操作的IOException。也可以自己继承Exception定义业务方面的异常。
  5. 非检查异常包括RuntimeException和Error。这部分不要求强制处理，很大可能性是代码问题，需要处理代码。
  6. 日常系统中，经常每个方法调用了别的方法，这个方法再调用别的，异常类型也不一样，为了避免异常信息丢失，一个异常中包含了导致它的异常的对象，这样可以通过cause属性取出原因，查看call stack。
  7. 异常发生的点后面代码就不会执行，直接进入catch。finally代码块保证了就算发生异常也会执行收尾工作，实际应用可以用来保证在异常发生时释放必要资源，比如JDBC中最后关闭connection, statement。
