Hashmap存储结构实现原理（JDK1.7及以前）

Posted on 2017-12-24 | Edited on 2018-05-13 | In Java集合类 | Views:

Map接口，被AbstractMap抽象类实现，HashMap继承自AbstratMap且实现Map接口。

public class HashMap<K,V> extends AbstractMap<K,V>
implements Map<K,V>, Cloneable, Serializable

Map接口可见定义了一个Entry接口：

/**
 * A map entry (key-value pair).  The <tt>Map.entrySet</tt> method returns
 * a collection-view of the map, whose elements are of this class.  The
 * <i>only</i> way to obtain a reference to a map entry is from the
 * iterator of this collection-view.  These <tt>Map.Entry</tt> objects are
 * valid <i>only</i> for the duration of the iteration; more formally,
 * the behavior of a map entry is undefined if the backing map has been
 * modified after the entry was returned by the iterator, except through
 * the <tt>setValue</tt> operation on the map entry.
 *
 * @see Map#entrySet()
 * @since 1.2
 */
interface Entry<K,V>

HashMap中有Node实现了Entry,并且具有NODE数组成员变量TABLE作为具体的存储结构。

数组：

/**
 * The table, initialized on first use, and resized as
 * necessary. When allocated, length is always a power of two.
 * (We also tolerate length zero in some operations to allow
 * bootstrapping mechanics that are currently not needed.)
 */
transient Node<K,V>[] table;

具体节点类：

/**
 * Basic hash bin node, used for most entries.  (See below for
 * TreeNode subclass, and in LinkedHashMap for its Entry subclass.)
 */
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;

    Node(int hash, K key, V value, Node<K,V> next) {
        this.hash = hash;
        this.key = key;
        this.value = value;
        this.next = next;
    }

    public final K getKey()        { return key; }
    public final V getValue()      { return value; }
    public final String toString() { return key + "=" + value; }

    public final int hashCode() {
        return Objects.hashCode(key) ^ Objects.hashCode(value);
    }

    public final V setValue(V newValue) {
        V oldValue = value;
        value = newValue;
        return oldValue;
    }

    public final boolean equals(Object o) {
        if (o == this)
            return true;
        if (o instanceof Map.Entry) {
            Map.Entry<?,?> e = (Map.Entry<?,?>)o;
            if (Objects.equals(key, e.getKey()) &&
                Objects.equals(value, e.getValue()))
                return true;
        }
        return false;
    }
}

查看Node的成员变量以及PUT方法可以得知：Hashmap放入元素时会先通过KEY计算HASHCODE并且2次加工（JDK8因为用了红黑树处理冲突能力加强所以2次处理较简单，高16位异或）来确定在TABLE数组中的位置，
新加的KEY对应的INDEX位置在TABLE时已经有值时，调用EQUALS方法比较，相等的话确定为同一个KEY，则替换原来值，相当于一个更新操作。
如果EQUALS不等则说明不是同一个KEY，是2个KEY得HASH值相同，即为哈希冲突，MODCOUNT+1结构改变，调用方法将这个节点加入此位置前一节点NEXT中。
即拉链法处理哈希冲突，哈希值相同不同NODE会在数组中处于同一个位置，形成一张单链表。

/**
 * Associates the specified value with the specified key in this map.
 * If the map previously contained a mapping for the key, the old
 * value is replaced.
 */
public V put(K key, V value) {
    if (table == EMPTY_TABLE) {
        inflateTable(threshold);
    }
    if (key == null)
        return putForNullKey(value);
    int hash = hash(key);
    int i = indexFor(hash, table.length);

    for (Entry<K,V> e = table[i]; e != null; e = e.next) {
        Object k;

        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) {
            V oldValue = e.value;
            e.value = value;
            e.recordAccess(this);
            return oldValue;
        }
    }

    modCount++;
    addEntry(hash, key, value, i);
    return null;
}
