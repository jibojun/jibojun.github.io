泛型基本使用

Posted on 2017-10-18 | Edited on 2018-07-18 | In JavaCore | Views:
  1. 泛型为JDK1.5开始的特性，就像声明方法时候指定参数一样，声明类，接口或者方法时，也可以指定一个类型参数，这个可以是变量，代表着类型可变。

  2. 泛型有以下主要好处：

    1. 避免了类型强转，之前使用如果是object需要强转，如果类型转换错误则会出class cast的异常
    2. 泛型只存在于编译期，与运行时（class文件生成，字节码）无关，运行期会被擦除，但编译期会直接报错，将错误提前了，提高了安全性。比如以下例子，一个set里加Long类型，如果加String，直接编译出错，IDE里可以直观看出。但如果用反射加入，运行时是可以的。

编译时报错：

 Set<Long> set=new HashSet();
set.add(1L);
set.add(2L);
//error when compile
set.add("ccc");

反射，可以插入其他类型元素：

         Set<Long> set=new HashSet();
        set.add(1L);
        set.add(2L);
        //error when compile
//        set.add("ccc");
        try {
            //reflect, get method and add string
            Class cls = Class.forName("java.util.HashSet");
            Method method=cls.getMethod("add",Object.class);
            method.invoke(set,"CCC");
            System.out.println(set);
        }catch (Exception e){
            e.printStackTrace();
        }
  1. Java中集合类例如List等都是使用泛型，所以可以兼容各种类型

  2. 基本使用例子，无通配符用法：

    public class Result<T> {
    private long id;
    private T response;

    public Result(long id, T response){
        this.id=id;
        this.response=response;
    }

    public long getId() {
        return id;
    }

    public void setId(long id) {
        this.id = id;
    }

    public T getResponse() {
        return response;
    }

    public void setResponse(T response) {
        this.response = response;
    }
}    

public class GenericTest {
public static void main(String[] args) {
            System.out.println(getResult(1).getResponse().getClass().getName());
            System.out.println(getResult(2).getResponse().getClass().getName());
            System.out.println(getResult(100).getResponse().getClass().getName());
        }

        private static Result getResult(int i){
            switch (i){
                case 1:
                    return new Result(1,"TEST");
                case 2:
                    return new Result(2,new ArrayList());
                default:
                    return new Result(3,new HashSet());
            }
        }
    }
